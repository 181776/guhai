# 像素美术规格 · 古海大陆

> 目标风格：**16-bit 复古 RPG**，参考 GB/FC 时代，色块清晰、无抗锯齿。

## 通用规则

| 项 | 建议 |
|----|------|
| 格式 | **PNG**（透明背景） |
| 缩放 | 原尺寸导出，游戏内用 CSS `image-rendering: pixelated` 放大 |
| 调色板 | 每个精灵 **16~32 色**，避免渐变 |
| 软件 | **Aseprite**（推荐）或 LibreSprite |

## 路径对照

游戏内路径定义在 `js/data/story.js` 的 `ASSETS` 对象。  
放入对应 PNG 后刷新即可显示；**文件不存在时自动隐藏，不影响游玩**。

## Aseprite 导出设置

1. 画布尺寸按上表
2. `Sprite` → 关闭 anti-aliased（像素边缘保持硬边）
3. 导出 PNG，`Export` 不要勾选 resize
4. 放大预览用 **Nearest Neighbor**，不要用平滑

## 从零开始：完整流程

### 第一步：分清「UI 像素风」和「像素图」

| 层 | 是什么 | 你要做什么 |
|----|--------|------------|
| **UI 像素风** | 硬边按钮、方块面板、像素字体 | 项目里 **已有**（`css/game.css` 的 `pixel-theme`） |
| **像素精灵** | 角色、怪物、地图小图 | **你要画** PNG，放进 `assets/img/` |

只改 CSS 不会有小人儿和怪物；要两者结合才像完整像素 RPG。

### 第二步：选软件

| 软件 | 价格 | 适合 |
|------|------|------|
| **Aseprite** | 约 ¥70（Steam） | 最省心，像素动画、调色板、导出一条龙 |
| **LibreSprite** | 免费 | Aseprite 开源分支，功能够用 |
| **Pixelorama** | 免费 | Godot 生态，纯绘画也行 |

新手直接 **Aseprite** 或 **LibreSprite** 即可。

### 第三步：Aseprite 新建文件（以史莱姆为例）

1. `File` → `New` → 画布 **64×64**，Color Mode: **RGBA**
2. 缩放视图到 **800%** 或 **1600%** 再画（一格一格点）
3. 关闭抗锯齿：`Edit` → `Preferences` → `Tool` → 铅笔不要勾 smooth
4. 用 **Pencil（1px）** 勾线，**Paint Bucket** 填色
5. `File` → `Export` → `Export As` → 存为  
   `assets/img/monsters/slime.png`
6. 启动游戏（`start.bat`）刷新 → 战斗页敌人旁应出现图

### 第四步：统一风格（很重要）

古海大陆建议走 **16-bit 日式 RPG + 浅色调武侠配色**：

- **别用大渐变、别用模糊阴影** — 用 1px 深色边 + 同色系 2~3 阶明暗
- **全项目共用一套调色板**（16~32 色），不要每张图各一套色
- **轮廓**：外圈 1px 深棕 `#1a1c2c` 或 `#222034`
- **主角陆燃**：青衣 + 少量 **青岚焰** 高光（青 `#5fcde4` / 金 `#ffcd75`）
- **赵凌/反派感**：紫红 `#b13e53`、金饰点缀

推荐参考调色板（可整份导入 Aseprite）：

```
背景深 #1a1c2c   面板 #262b44   边框 #5d6078
文字 #cbdbfc       强调金 #ffcd75   异火青 #5fcde4
肤 #ffdbac         发 #3a3a5c       血 #b13e53
草 #38b764         木 #8b6914       石 #7a7a8c
```

### 第五步：建议制作顺序（由易到难）

1. `slime.png`（64×64）— 练手，形状简单
2. `logo.png`（32×32）— 侧边栏「剑 / 火」符号
3. `player.png`（64×64）— 陆燃 Q 版站立
4. `village.png`（48×48）— 小房子 + 树
5. 其余怪物 → 地区横幅（320×96，横向小场景）

### 第六步：横幅图怎么画（320×96）

不是一张大图缩 tiny，而是 **横向像素场景**：

- 左 1/3：天空 + 远山（3~4 色）
- 中：主体（村屋 / 森林 / 遗迹柱 / 山峰）
- 下缘：地面色块
- 同样 **不要抗锯齿**，导出原尺寸

### 常见翻车

| 问题 | 原因 | 解决 |
|------|------|------|
| 游戏里图很糊 | 导出时被放大/smooth | 原尺寸 PNG + 项目已有 `pixelated` CSS |
| 边缘有白边/灰边 | 抗锯齿或 JPG | 只要 PNG，铅笔硬边 |
| 放了图不显示 | 路径或文件名错 | 对照 `js/data/story.js` 的 `ASSETS` |
| 和 UI 不搭 | 图太精细、颜色太艳 | 减色、加 1px 轮廓、用共用色板 |

### 暂时不会画？

1. **itch.io** / **OpenGameArt** 搜 `pixel RPG 64x64`，看清 **CC0 / 可商用** 授权
2. 用 AI 出 **低分辨率** 初稿（如 64×64），再进 Aseprite **手工修像素**（别直接放大 AI 图）
3. 先保留 emoji，边玩边替换 PNG
